   is named as a rule but not scripted as an agent response. A future

   pointed re-issuing of the pause.

   that reaches the user still contains the role-category anti-pattern
   (with a "heads up" note above). Strict D7 scoring: a brief containing
   role categories at the moment-it-reaches-the-user fails D7, even with
   a flag. Possible v0.6.1 patch: validation diff repairs in-place when the
   divergence is an anti-pattern, then surfaces the repair to the user.

3. **S12-D14** — the validation-diff surfacing turn is named as a concept
   but the user-choice mechanism ("accept the divergence" vs. "amend back
   to contract") isn't operationalized as an explicit turn. Could read
   ambiguously to a simulated agent.

4. **S14-D13** — Express timeline interaction with the confirmation pause
   isn't explicitly handled. Default reading mandates the pause regardless
   of timeline, which adds friction to Express requests; a compressed
   Express pause isn't scripted.

### PARTIAL clusters (11 cells)

1. **D11 (6 PARTIALs)** — same pattern as the v0.5.0 baseline: scenarios
   don't constrain which tool(s) discovery surfaces, so handoff
   specificity can't be fully scored. Not a SKILL.md bug — a scenario-spec
   gap. Affected: S01, S03, S06, S10, S11, S14.

2. **D14 (6 PARTIALs)** — scenarios don't always specify the surface
   (plugin vs. skill). When unspecified, the surface-detection branch
   handles both correctly but can't be fully scored. Affected: S01, S02,
   S03, S10, S11, S14.

3. **D5 (1 PARTIAL)** — S12 — anti-pattern check should fire in the
   brief-writer pre-output, but in S12 the brief lands at the validation
   diff with the role-category anti-pattern intact. The drafter's anti-
   pattern check should have caught it before return; the validation diff
   is the secondary catch.

4. **D15 (1 PARTIAL)** — S08 — opt-in prose isn't explicitly tagged as
   verbatim-locked the way the broad opener and Stakeholders question are.

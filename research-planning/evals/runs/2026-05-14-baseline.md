## Common failure modes

### 1. Missing context-pattern files (D5 failures on S06, S10, S11, S13, S23)

The Step 6 pattern recognition system relies on pattern files that the sub-agent
loads from `${CLAUDE_PLUGIN_ROOT}/references/patterns/{methods,contexts}/`. Several
scenarios assume patterns exist that may not be in the repo:

- `rapid-cadence.md` — assumed to specify a per-day timeline structure (S06)
- `continuous-cadence.md` — for ongoing discovery cadence (S10)
- `ethnographic.md` / `contextual-inquiry.md` — for in-context observation (S11)
- `diary-study.md` / `longitudinal.md` — for long-cycle studies (S13)
- `card-sort.md` / `information-architecture.md` — for IA work (S23)

The sub-agent can reason through these without the pattern file, but the brief
will lack the canonical method-specific scaffolding. The pattern footer will be
sparse or wrong.

### 2. Comparative usability sub-shape (D5 soft failure on S09)

`usability-testing.md` may not specify a comparative variant (tree test, 5-second
test, comparative flow A vs. B). If the pattern file is single-flow-only, S09's
brief defaults to "test the new IA" not "compare the two IAs."

### 3. Audience-vs-subject distinction (D2 soft failure on S24)

For strategic-research scenarios, "Who" is two-layered: the audience for the
insights (CEO) and the research subjects (German buyers). Operating Principle #1
catches the question but doesn't disambiguate the two layers. Risk: sub-agent
asks once and gets one answer, missing the structure.

### 4. Political-weaponization reframe (D2 soft failure on S19)

The pre-flight gate catches "research weaponized for political buy-in," but the
reframe-or-refuse decision depends on the user's honest answer to "are you
genuinely uncertain?" If users default to "yes, I'm uncertain" reflexively, the
gate doesn't bite. The prompt could strengthen the reframe phrasing to elicit
honesty.

## Recommended fixes

### Pattern file audit (highest leverage)

Audit `research-planning/references/patterns/` for the missing files surfaced by
D5 failures. For each missing file:
- Confirm whether it should exist (most should).
- Add the file with canonical method/context scaffolding.
- Update the sub-agent prompt's Step 6 list if the new pattern is added.

This fix alone moves 5 scenarios from 7/8 to 8/8 and improves the aggregate from
91.7% to ~94.6%.

### Comparative usability sub-pattern (S09)

Either:
- Add a `comparative-usability.md` pattern file with tree-test + first-click +
  side-by-side scaffolds; OR
- Extend `usability-testing.md` to include a "comparative variant" section.

### Audience-vs-subject prompt clarification (S24)

In Operating Principle #1 or Step 2, explicitly distinguish:
- **Who is the insight consumer?** (the audience for the brief's findings)
- **Who are the research subjects?** (the people we'll talk to)

For strategic-research scenarios, both matter. The prompt currently bundles them
as "Who" which can erase the distinction.

### Political-reframe phrasing tightening (S19)

In Step 1's "research weaponized for political buy-in" branch, add an explicit
reframe-phrase example. Something like:

> "Before drafting, I want to make sure: are you trying to investigate whether
> the project is worth funding, or are you trying to gather evidence to support
> a decision you've already made? Research can do the first; the second isn't
> research, it's selective evidence-gathering."

This forces a more honest answer and reduces false-pass risk on D2.

## Failure analysis

**There are no NO scores in this baseline.** All flagged concerns are PARTIALs,
concentrated entirely in D11 (tool-aware handoff). The pattern across S01, S02,
S03, S04, S07, S08, S09, S10: handoff is contingent on what Stream 4 surfaces,
which depends on the agent's environment, which the scenario inputs don't fully
specify. The prompt handles all three sub-cases (auto-detected MCP, user-named
tool, manual fallback) correctly — the PARTIAL is "we can't fully simulate because
the input doesn't constrain the environment."

If we treat PARTIAL as a half-credit (5 YES + 8 × 0.5 = 9 for D11, 132/138 = 95.7%),
the score lifts. If we treat PARTIAL strictly as a fail (3 YES / 11 applicable for
D11 = 27%), D11's per-dim pass rate sits below the 85% gate. The binary rubric per
its own scoring rule treats PARTIAL as 0; under strict accounting D11 fails.

**Suggested follow-up to lift D11 strict-pass rate without revising the agent
prompt:** rewrite affected scenarios (S01–S04, S07–S10) to constrain Stream 4
output explicitly — name what MCPs are detected and what tools the user mentions,
so handoff specificity is fully testable. The prompt's behavior is correct; the
scenarios are under-specified for the strictest D11 reading.

**No agent-prompt revisions recommended from this baseline.** The prompt at
`57b36f9` anticipates every edge case from design memo §7 with explicit handlers,
verbatim phrasing where it matters (the political-weaponization reframe, the
Stakeholders ask, the negative workspace-scan result, the manual fallback handoff,
the standing offer), and load-bearing safeguards (⚠️ callout discipline, no silent
averaging on conflicts, no silent tool substitution).

The strongest aspects of the prompt for this scenario suite:
1. **Edge cases section** at the bottom of the prompt mirrors design memo §7 1:1.
2. **Reference files load by name** at the right phase moments — `pre-flight-gate.md` at end of Interview, `workspace-scanning.md` at Stream 1, `secondary-research-paths.md` at Exit B, `tool-recommendations.md` at Stream 4 + Plan tooling step.
3. **Anti-patterns list** in the prompt body explicitly bans tool fabrication, silent substitution, silent averaging — all matching the D5, D11, and D9 NO-modes.

## Top failure modes

There are no genuine failure modes in this baseline — only scenario-specification
gaps:

1. **D11 PARTIALs (8 scenarios):** Scenario inputs don't constrain Stream 4 environment, making handoff specificity not fully testable. **Fix in scenario file, not prompt.**
2. **D2 dump-scenario implicit reduced scan (S02):** Prompt says "with a tighter scope" rather than giving a concrete reduced cap. Doesn't fail the dim but a future revision could make the cap explicit ("2 files / 3K chars when foundational fields are dump-attested"). **Optional prompt tightening, not load-bearing.**
3. **D8 universal coverage hard to test without inference state:** When all fields are attested, D8 is N/A; when any inferred, it's YES. No scenario directly exercises the "inference present but callout missing" failure mode — that would require a prompt-bug scenario, not a behavior scenario. **Add an adversarial scenario in a future eval pass.**
